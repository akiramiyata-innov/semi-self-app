# Changelog

## [1.7.7] - 2026-07-09

### 修正
- キオスクの2カメラが Edge で正面/手元が入れ替わる問題を修正
  - 原因：`enumerateDevices()` の並び順がブラウザ依存（Chrome と Edge で逆）で、順番で割り当てていたため
  - 対策：ブラウザ間で同一の label でソートしてから割り当て、どのブラウザでも同じ物理カメラが正面/手元になるよう固定
  - 併せて、設置の都合で入れ替えたい場合の `?swapcam=1` URLパラメータを追加
- キオスクの「キャンセル」ボタン（通話終了）が効かない問題を修正（v1.7.1 のリグレッション）
  - 原因：v1.7.1 で追加した call:end のガードが「担当スタッフ以外は終了不可」となっており、利用者側の終了までブロックしていた
  - 対策：セッションの担当スタッフ or 利用者本人なら終了できるよう変更（第三者スタッフによる終了は引き続きブロック）

## [1.7.6] - 2026-07-07

### 変更（スタッフ画面・管理メニューの調整）
- スタッフ画面アカウントメニューの「通話ログ」「用語集管理」を、表示したままクリック不可（薄いグレー）に変更
  - 表示フラグ `CALL_LOGS_ENABLED` / `GLOSSARY_ADMIN_ENABLED`（既定 false）を追加。`true` にすれば通常リンクに戻る
- スタッフ画面の管理者メニューに「駅マスター登録」（/admin/stations）を追加
- スタッフ管理画面のナビゲーションから「用語集」ボタンを削除

## [1.7.5] - 2026-07-07

### 追加・変更（スタッフ画面のUI改善）
- ログイン中のアカウント（メールアドレス）を表示：表示名とは別に、どのアカウントでログイン中かを確認できるように
  - ヘッダーのアカウントメニュー内、および「パスワード変更」パネルに表示
- スタッフ画面ヘッダーを整理（管理者で最大13個並んでいたボタンを2つのメニューに集約）
  - アカウントメニュー（👤）：アカウント情報／マイク許可確認／通話ログ／担当駅設定／パスワード変更／（管理者）スタッフ管理・用語集管理／ログアウト
  - キオスクメニュー：デモ用キオスク画面の起動（券売機1番・2番・精算機1番）
  - 常時表示はステータス・オンライン人数・接続状態のみに簡素化。各メニューはメニュー外クリックで閉じる

## [1.7.4] - 2026-07-07

### 変更（コードレビュー指摘対応・バッチ3-b／挙動は不変のリファクタ）
- Next.js 16 で非推奨になった `middleware` 規約を `proxy` に移行（`middleware.ts` → `proxy.ts`、関数名 `middleware` → `proxy`）。起動時の非推奨警告を解消
- 認証チェックの重複を解消：各 API ルートにコピペされていた `requireAdmin` / `requireAuth` を `lib/session.ts` の `requireAdmin` / `getSessionFromRequest` に一本化（11ファイル）
- GCS＋ローカルファイル＋TTLキャッシュの重複実装（用語集・駅・担当駅の3モジュール）を汎用の `lib/jsonStore.ts`（`createJsonStore`）に集約。`getGCSBucket` も `gcsClient` から再利用
- 未使用のデッドコード（`stations/import` の `normalize` 関数）を削除
- 正味 約180行削減（挙動は不変・認証レスポンス／Fresh読み取りの検証済み）

## [1.7.3] - 2026-07-07

### 修正（コードレビュー指摘対応・バッチ3-a）
- 管理画面での用語集・担当駅の変更が通話サーバーに最大5分間反映されない問題を修正（キャッシュ二重化）
  - 通話サーバー（tsx）とNext APIルートは別モジュールインスタンスで、片方のキャッシュ無効化がもう片方に届かないのが原因
  - `getGlossaryTermsFresh()` / `getAssignmentsFresh()`（キャッシュ非経由の読み取り）を追加し、通話サーバーはこれを使って毎回最新を読むよう変更（A案：即反映優先）
  - Next側（管理画面・STT）は従来どおりキャッシュ維持

## [1.7.2] - 2026-07-07

### 修正（コードレビュー指摘対応・バッチ2）
- 未使用かつ認証なしで公開されていた `/api/translate`・`/api/tts` を削除（有料Google APIの外部悪用を防止。実処理は socketServer 内にあり参照ゼロを確認）
- 繁体字（zh-TW）ユーザーにエラー画面が日本語で表示される問題を修正（ERRメッセージ表に台湾繁体字を追加）
- 担当駅ルーティングの抜け穴を修正
  - 判定を `isStaffEligible` / `isStaffAssignedToStation` に一本化
  - `staff:join` のキュー再送を担当駅でフィルタし、再接続・再読込時に担当外の駅の着信が届かないよう修正
  - `call:answer` に担当駅ガードを追加、クライアントは担当駅の読込後にのみ `staff:join` するよう変更（空登録によるフェイルオープンを防止）
- Excel一括登録で数値セルがあると取り込み全体が失敗する問題を修正（`raw: false` で全セルを文字列化・用語集/スタッフ/駅マスターの3画面共通）

## [1.7.1] - 2026-07-07

### 修正（コードレビュー指摘対応）
- 通話ログAPI（`/api/logs`・`/api/logs/[sessionId]`）に認証を追加。未ログインでは401を返すよう修正（従来は誰でも全通話記録を取得可能だった）
  - `lib/session.ts` に共通の `getSessionFromRequest` ヘルパーを追加
- 再接続時に前の通話状態（会話・音声・マイク・セッションID）を確実にクリアするよう修正。共用キオスクで前のお客様の会話が次のお客様に残らないよう `resetCallState` に統一
- 応答レースで負けたスタッフ側に幽霊通話パネルが残る問題を修正。加えてサーバー側に「セッション担当スタッフ以外は発話・終了できない」ガードを追加

## [1.7.0] - 2026-07-06

### 追加
- キオスク側カメラを正面（顔）・手元の2台構成に対応
  - `hooks/useScreenCapture.ts` が特定のカメラ機器（`deviceId`）を指定して撮影できるように拡張
  - `app/user/UserScreen.tsx` で接続カメラを自動検出し、1台目＝正面／2台目＝手元として2系統同時に撮影・配信
  - スタッフ画面（`ActiveCallPanel`）に正面・手元を上下2段で表示（手元をより大きく表示）
- 着信中（応答前）の顔カメラプレビューに対応
  - `call:requested` イベントを追加し、呼び出しと同時にセッションIDをユーザー側へ通知（従来は応答後まで不明だった）
  - 応答前の顔カメラ映像を担当駅の対象スタッフ全員へ中継し、着信カードの横にサムネイル表示
  - 「応答」時に着信中のプレビュー映像をそのまま通話画面へ引き継ぎ（手元カメラは応答後に起動）
